# Interactive Graphics

## Установка пакетов

Скопируйте и выполните в R/RStudio:

```r
install.packages(c(
  "shiny",
  "Rcpp",
  "Rssa",
  "lattice",
  "gridExtra",
  "dplyr",
  "imager",
  "reticulate",
  "ggplot2",
  "stringi",
  "BiocManager"
))

BiocManager::install("EBImage", ask = FALSE, update = FALSE)
```

Для BM3D дополнительно нужен Python-модуль:

```r
reticulate::py_install("bm3d", pip = TRUE)
```

Если `bm3d` не установлен, приложение все равно запускается, просто метод BM3D
будет недоступен.


## 1. HT Error App

Путь к приложению:

`interactive_graphics/ht_error_app/app.R`

Сравниваются методы:

- `cssa_row_row`;
- `median`;
- `wiener`;
- `bm3d`.

Для каждого метода оцениваются ошибки по параметрам `rho` и `theta`.

Приложение поддерживает два варианта HT:

- обычное преобразование Хафа;
- weighted HT.

В интерфейсе можно менять основные параметры эксперимента:

- размер матрицы;
- уровень шума;
- число прямых;
- параметры прямых;
- порог для выделения значимых точек.
- число потоков для параллельного расчета независимых шумовых реализаций.

Также в приложении есть вкладка `Find big error`.

Она нужна для поиска интересных случаев, когда ошибка получается сильно больше идеальной. Обычно такие ситуации возникают из-за дискретизации: прямая в непрерывной модели есть, но на дискретной сетке она может отображаться неидеально.

Запуск из R:

```r
shiny::runApp("interactive_graphics/ht_error_app")
```

## 2. Frequency Error App

Путь к приложению:

`interactive_graphics/frequency_error_app/app.R`

Это приложение показывает, как ошибка локализации зависит от частоты `omega_j`.


В приложении можно настраивать:

- уровень шума;
- размер выборки;
- размер вектора.
- число потоков для параллельного расчета повторов.

Запуск из R:

```r
shiny::runApp("interactive_graphics/frequency_error_app")
```
